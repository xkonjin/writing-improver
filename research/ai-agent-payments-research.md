- **Product**: AI Agent Market - decentralized on-chain marketplace

- **Security**: Shade Protocol using TEEs (Trusted Execution Environments) for Shade Agents - critical for DeFi-based AI agents handling funds
- **2026 Position**: Leading AI-friendly blockchain driven by high throughput, fast finality
  - Source: [AMINA Bank](https://aminagroup.com/research/near-protocol-bringing-ai-and-smart-ux-onchain/)

## 8. IDENTITY/AUTHENTICATION FOR AI AGENTS

### Know Your Agent (KYA) Framework

**Definition**

- Framework for verifying, governing, and holding AI agents accountable
- Ensures actions are authorized, auditable, and bound to trusted identities
- Like KYC for customers, KYB for businesses
- Answers: Who made this agent? Who does it represent? What can it do?
- Source: [Sumsub](https://sumsub.com/blog/know-your-agent/)

**Core Components**

1. **Identity Verification**
   - Agent registries and trusted identity schemas
   - Agents registered under known identity signed by developers or platforms
   - Confirms authorship, provider, version control
   - Source: [AstraSync AI](https://medium.com/@astrasyncai/know-your-agent-kya-establishing-the-standard-for-ai-agent-identity-and-trust-d0fb779fc657)

2. **Authentication & Authorization**
   - Process of verifying agents across identity, authentication, authorization, and policy enforcement
   - Source: [Trustfull](https://trustfull.com/articles/know-your-agent-how-to-verify-ai-agents-at-scale)

3. **Scope and Permission Mapping**
   - Know what agent is allowed to do
   - Track declared intent against actual behavior
   - Source: [Know Your Agent Network](https://knowyouragent.network/)

**Market Status**

**Current State**

- KYA procedures NOT yet widely standardized or formally established across industries
- Early frameworks and discussions emerging
- Mainly in organizations experimenting with AI agents and autonomous systems
- Source: [Vouched](https://www.vouched.id/learn/blog/building-a-trust-framework-for-ai-agents-why-know-your-agent-matters)

**Market Players**

- "The Know Your Agent market now has a dozen players"
- Key participants: Visa, Trulioo, Billions Network, Vouched, wave of startups
- All claiming to solve "agent identity"
- Source: [Know Your Agent Network](https://knowyouragent.network/)

### Emerging Standards

**ERC-8004**

- Ethereum's "Trustless Agents" standard
- Establishes blockchain-native identity infrastructure for AI agents
- Features:
  - Agents discover each other
  - Build portable reputation
  - Transact across organizational boundaries
- Source: [Billions Network](https://billions.network/blog/billions-know-your-agent)

**Skyfire KYAPay Protocol**

- Know Your Agent Pay protocol
- Used in December 2025 demonstration with Visa Intelligent Commerce

## 10. PAYMENT PROTOCOL STANDARDS

### Agentic Commerce Protocol (ACP)

**Maintainers**

- OpenAI and Stripe
- Released 2025
- Currently in beta
- Source: [GitHub - Agentic Commerce Protocol](https://github.com/agentic-commerce-protocol/agentic-commerce-protocol)

**Core Architecture**

- RESTful interface OR MCP server implementation
- Defines common language for how agents and businesses transact
- Coordinates checkout and securely shares payment credentials
- Source: [Stripe Documentation](https://docs.stripe.com/agentic-commerce/protocol)

**Key Endpoints** (4 required):

1. Create Checkout object: Generate cart and checkout data from SKU
2. Update Checkout object: Modify quantities, fulfillment, customer details
3. Complete Checkout object: Pass SharedPaymentToken, process payment
4. Cancel Checkout object: Release inventory, update status

- Source: [OpenAI Developer Docs](https://developers.openai.com/commerce/guides/get-started/)

**Security Specifications**

- Requires HTTPS
- Authorization: Bearer {token}
- All webhook events must be signed with HMAC signature in request header
- Must use PSP compliant with Delegated Payment Spec
- Stripe's Shared Payment Token: First Delegated Payment Spec-compatible implementation
- Source: [Stripe Blog](https://stripe.com/blog/developing-an-open-standard-for-agentic-commerce)

**Specification Versions**

- 2025-09-29: Initial release
- 2025-12-12: Fulfillment enhancements
- 2026-01-16: Capability negotiation
- 2026-01-30: Extensions, discounts, payment handlers
- Source: [GitHub - Agentic Commerce Protocol](https://github.com/agentic-commerce-protocol/agentic-commerce-protocol)

**Governance**

- Open source, community-designed
- Apache 2.0 license
- Jointly governed by OpenAI and Stripe as Founding Maintainers
- Clear path toward broader community governance
- Source: [Agentic Commerce Dev](https://www.agenticcommerce.dev/)

### Agent Payments Protocol (AP2)

**Developer**

- Google (with leading payments and technology companies)
- Announced 2025
- Source: [Google Cloud Blog](https://cloud.google.com/blog/products/ai-machine-learning/announcing-agents-to-payments-ap2-protocol)

**Core Features**

- Open, shared protocol
- Common language for secure, compliant transactions between agents and merchants
- Prevents fragmented ecosystem
- Supports multiple payment types:
  - Credit and debit cards
  - Stablecoins (via x402 extension)
  - Real-time bank transfers
- Source: [Zuplo Blog](https://zuplo.com/blog/mcp-api-payments-with-x402)

**Mandates System**

- Uses "Mandates" - tamper-proof, cryptographically-signed digital contracts
- Serve as verifiable proof of user's instructions
- Signed by verifiable credentials (VCs)
- Act as foundational evidence for every transaction
- Source: [Orium](https://orium.com/blog/agentic-payments-acp-ap2-x402)

**Extension Ecosystem**

- x402: First extension to AP2, only stablecoin facilitator
- Source: [Coinbase Developer Platform](https://www.coinbase.com/developer-platform/discover/launches/google_x402)

### x402 Protocol

**Specification**

- Stablecoin payment extension for AP2
- Named after HTTP status code 402 ("Payment Required")
- Source: [Eco Support](https://eco.com/support/en/articles/12328618-what-is-agent2agent-x402-the-complete-guide-to-ai-agent-payment-protocol)

**Functionality**

- Enables instant stablecoin payments directly via HTTP
- Facilitates pay-per-use access to APIs, data, digital services
- Makes settlement programmable and reactive
- Agents can pay each other for data, GPU time, API calls instantly and permissionlessly
- Source: [Medium - Jung-Hua Liu](https://medium.com/@gwrx2005/ai-agents-and-autonomous-payments-a-comparative-study-of-x402-and-ap2-protocols-e71b572d9838)

**Integration Support**

- Coinbase's AgentKit
- Cloudflare Workers
- Vercel's x402-mcp
- Source: [Coinbase Developer Platform](https://www.coinbase.com/developer-platform/discover/launches/google_x402)

**Status**

- Production-ready as of 2025
- Brian Armstrong (Coinbase CEO): "x402 + Google just unlocked a new level for AI agents. Agents can actually pay each other now, with x402 powering the stablecoin rail inside Google's new Agentic Payments Protocol (AP2)."
- Source: [Brian Armstrong on X](https://x.com/brian_armstrong/status/1967980850567582096)
